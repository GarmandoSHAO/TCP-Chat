TCP 聊天室 — 打包与安装包方案（图标统一版）
日期: 2026-06-22 | 版本: 2.0（图标驱动重构） 定位: 整合 PyInstaller 打包 + Inno Setup 安装包，以 TCP-Chat.png 为图标唯一信源

1. 方案概述
以 TCP-Chat.png 作为图标唯一信源（Single Source of Truth），构建一条完整的从源码 → EXE → 安装包的交付流水线。所有图标（EXE 文件图标、安装包图标、窗口标题栏图标、任务栏图标）均从此 PNG 生成，保证视觉完全一致。

2. 现状分析
当前项目已有打包体系，但存在图标信源割裂问题：

环节	当前实现	图标来源	问题
PyInstaller .spec	icon='TCP-Chat.ico'	.ico 文件	ICO 需手动维护，可能与 PNG 不一致
Inno Setup .iss	SetupIconFile=..\TCP-Chat.ico	.ico 文件	同上
运行时窗口图标	icon_manager.py 用 set_window_icon()	PNG + ICO 双加载	icon_manager.py 有自动重建机制，但构建流水线中缺少构建时 ICO 刷新步骤
build_installer.bat	按步执行 PyInstaller → 复制工具 → ISCC	无图标重建步骤	构建时不会自动用最新 PNG 更新 ICO
结论：虽然后续的 2026-06-22-窗口与EXE图标方案.md 解决了运行时图标问题，但构建流水线中缺少「构建时自动从 PNG 生成 ICO」的步骤，且方案分散在两份文档中。

6. 方案 A（推荐）：构建时 + 运行时双保险
维度	说明
图标信源	TCP-Chat.png（唯一）
构建时	build_icon.py 在 PyInstaller 前运行，自动生成多尺寸 ICO
运行时	icon_manager.py 的 ensure_ico() 兜底：如果 ICO 缺失或过期则重建
EXE 图标	PyInstaller icon='TCP-Chat.ico'
安装包图标	Inno Setup SetupIconFile=..\TCP-Chat.ico
窗口图标	set_window_icon() 双 API：iconphoto(PNG) + iconbitmap(ICO)
风险等级	低
开发成本	低（新增 tools/build_icon.py 约 50 行，修改 build_installer.bat 约 5 行）
维护成本	极低——替换 TCP-Chat.png 后重建即可，一切自动
优点：

构建时确保 ICO 最新，EXE 和安装包图标与 PNG 完全一致
运行时兜底，即使构建时漏掉（如开发环境直接跑源码）也能正常工作
所有图标环节共享同一 PNG 源，不再有"ICO 与 PNG 不一致"的问题
缺点：

增加一个构建步骤（~0.5 秒开销）
构建机器需要安装 Pillow（通常项目已依赖）
7. 方案 B（备选）：仅运行时自动重建
维度	说明
图标信源	TCP-Chat.png
构建时	不新增步骤，沿用当前 TCP-Chat.ico（需手动维护）
运行时	icon_manager.py 兜底重建
风险等级	中
适用场景	不希望改动构建流水线
优点：不修改构建脚本，改动最小

缺点：

构建时的 ICO 可能不是最新 PNG 生成的 → EXE 文件图标和安装包图标可能与 PNG 不同
只有用户运行程序后，运行时重建才会修正窗口图标，但 EXE 文件图标是固定的
如果构建者忘记手动更新 ICO，产出的安装包图标就是旧的
8. 方案 C（备选）：纯构建时生成，运行时无 Pillow
维度	说明
图标信源	TCP-Chat.png
构建时	build_icon.py 生成 ICO，且将 ICO 所有尺寸嵌入 EXE
运行时	icon_manager.py 只加载 ICO，不依赖 Pillow
风险等级	低（但需要移除运行时动态重建）
适用场景	希望缩小打包体积（不打包 Pillow）
优点：

运行时不需要 Pillow，可减少打包体积（Pillow 约 3-5 MB）
减少运行时启动开销
缺点：

运行时如果 ICO 文件损坏或丢失，无法自动修复
用户替换 TCP-Chat.png 后需要重新打包才能生效（运行时不再自动重建）
需要修改 icon_manager.py 去掉 Pillow 相关逻辑

13. 推荐的 build_icon.py 伪实现（仅供接口参考）
#!/usr/bin/env python3
"""构建时图标生成工具：从 TCP-Chat.png 生成多尺寸 TCP-Chat.ico"""

import argparse, os, sys
from PIL import Image

DEFAULT_SIZES = (256, 128, 64, 48, 32, 24, 16)

def build_icon(png_path: str = "TCP-Chat.png",
               ico_path: str = "TCP-Chat.ico",
               sizes: tuple[int, ...] = DEFAULT_SIZES,
               force: bool = False) -> bool:
    if not os.path.exists(png_path):
        raise FileNotFoundError(f"源 PNG 不存在: {png_path}")
    if not force and os.path.exists(ico_path):
        png_mtime = os.path.getmtime(png_path)
        ico_mtime = os.path.getmtime(ico_path)
        if ico_mtime >= png_mtime:
            return True  # 已最新，跳过
    with Image.open(png_path) as src:
        frames = [src.resize((s, s), Image.Resampling.LANCZOS).convert("RGBA") for s in sizes]
        frames[0].save(ico_path, format="ICO", append_images=frames[1:])
    return True

if __name__ == "__main__":
    ap = argparse.ArgumentParser(description="从 PNG 生成多尺寸 ICO")
    ap.add_argument("--input", default="TCP-Chat.png")
    ap.add_argument("--output", default="TCP-Chat.ico")
    ap.add_argument("--force", action="store_true", help="强制重建")
    args = ap.parse_args()
    try:
        build_icon(args.input, args.output, force=args.force)
    except FileNotFoundError as e:
        print(f"[FAIL] {e}", file=sys.stderr)
        sys.exit(1)

14. 总结
本方案以 TCP-Chat.png 为唯一图标信源，通过构建时 build_icon.py 生成多尺寸 ICO，串联 PyInstaller（EXE 图标）和 Inno Setup（安装包图标），运行时 icon_manager.py 兜底重建，形成完整的图标保障链路。

推荐方案 A：构建时 + 运行时双保险，风险低、维护成本低，用户只需替换 TCP-Chat.png 即可更换整套软件图标。
