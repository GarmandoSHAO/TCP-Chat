# 窗口左上角图标 + EXE 文件图标方案

> 日期：2026-06-22 | 方案：A（推荐） | 风险：低

## 1. 需求概述

- 软件窗口左上角（标题栏）图标显示为 `TCP-Chat.png` 的图像
- PyInstaller 打包 → Inno Setup 安装后的 `.exe` 文件图标也是同一图像

## 2. 现状诊断

| 场景 | 当前状态 | 是否生效 |
|------|---------|---------|
| EXE 文件图标 | `.spec` 已配置 `icon='TCP-Chat.ico'` | ✅ |
| 安装包自身图标 | `.iss` 已配置 `SetupIconFile=..\TCP-Chat.ico` | ✅ |
| 窗口标题栏图标 | 代码仅调用 `iconphoto()` 加载 PNG | ⚠️ 任务栏有效，标题栏不可靠 |
| 运行时 PNG 可用性 | `.spec` 中 `datas=[]` 为空，PNG 未被 PyInstaller 打包，靠 bat 脚本手动 copy | ❌ 脆弱的旁路方案 |

### 根因

1. **标题栏图标**: Windows 下 `iconphoto()` 主要作用于任务栏，标题栏图标需靠 `iconbitmap()` + `.ico` 文件设置
2. **打包缺失**: `datas=[]` 未声明图标文件，运行时找不到文件 → 图标加载失败 → 静默回退到内置绿色方块

## 5. 接口设计（`_set_window_icon` 重构）

```
def _set_window_icon(self) -> None:
    """
    设置窗口图标：标题栏用 .ico（iconbitmap），任务栏用 .png（iconphoto）

    加载顺序:
      1. iconbitmap(TCP-Chat.ico)  →  标题栏 + Alt+Tab
      2. iconphoto(TCP-Chat.png)  →  任务栏（高 DPI）
      3. fallback: 内置 16×16 绿色方块

    异常策略: 每一步均被 try/except 包裹，加载失败不影响程序启动
    """
```

**关键点**:
- `iconbitmap()` 在前 —— 设置 Windows 原生标题栏图标
- `iconphoto()` 在后 —— 用更高分辨率的 PNG 覆盖任务栏图标
- 两个文件路径都通过 `get_app_root()` 定位，覆盖源码运行和打包运行两种场景

## 7. 风险评估与备选方案

### 方案 A（推荐）：双 API 调用，ICO + PNG 均打包

- **风险**: 低
- **描述**: `iconbitmap()` 设标题栏图标，`iconphoto()` 设任务栏图标，两文件均通过 `datas` 正式打包
- **优点**: Windows 标题栏+任务栏双保险，高 DPI 友好
- **缺点**: 需要 assurance 两个图标文件视觉一致

### 方案 B：仅用 iconbitmap()，统一用 ICO

- **风险**: 低
- **描述**: 只调用 `iconbitmap()` 设 `.ico`，去掉 `iconphoto()` 调用
- **优点**: 代码更简洁
- **缺点**: `iconbitmap()` 对高 DPI 任务栏支持不如 `iconphoto()`，大显示器上任务栏图标可能模糊

### 方案 C：仅用 iconphoto() + 确保 PNG 打包

- **风险**: 中
- **描述**: 保持现有代码不变，只修复 `datas=[]` 确保 PNG 被打包
- **优点**: 改动最小
- **缺点**: Windows 下 `iconphoto()` 对标题栏图标的设置不够可靠，这正是用户当前遇到的问题

## 9. 实施后纠错（2026-06-22）

### 问题 1（已修复）：iconbitmap / iconphoto 调用顺序

**现象**: 按方案 A 初次实施后（iconbitmap → iconphoto），运行 main.py 窗口图标未变化。

**根因分析**: Windows tkinter 图标 API 行为：

| API | 底层机制 | 作用位置 |
|-----|---------|---------|
| `iconbitmap(ico)` | `SetClassLong(hwnd, GCL_HICON)` + `WM_SETICON` | 窗口类图标 + 窗口图标 |
| `iconphoto(png)` | `WM_SETICON`（从 PNG 生成图标） | 窗口图标 |

**关键发现**: `WM_SETICON` 设置的窗口图标优先级**高于**窗口类图标。当 `iconphoto()` 在 `iconbitmap()` 之后调用时，`iconphoto()` 的 `WM_SETICON` 会**覆盖** `iconbitmap()` 设置的所有窗口图标，导致标题栏仍显示 PNG 生成的图标（对于 64×64 PNG 转出的 16×16 标题栏图标可能不兼容）。

**修复**: 调换调用顺序：`iconphoto()` 先 → `iconbitmap()` 后。

```
iconphoto(png) → WM_SETICON（PNG 兜底）
    ↓
iconbitmap(ico) → WM_SETICON（多尺寸 ICO 覆盖）+ SetClassLong（标题栏）
    ↓
最终生效: ICO 的多尺寸图标（16×16 标题栏 + 32×32 任务栏）
```

### 问题 2（已修复）：InitialInterface 的 Toplevel 独立窗口

**现象**: 主窗口（`root`）图标正确，但用户一启动看到的初始界面窗口图标为默认 tkinter 图标。

**根因分析**:

```
ChatClientUI.__init__()
  ├─ _set_window_icon(self.root)    ← root 图标 ✅
  └─ after(200, _open_initial_interface)
       └─ InitialInterface.show()
            └─ CTkToplevel(self.root)  ← 独立 Toplevel，不继承父窗口图标 ❌
```

`InitialInterface` 在 `show()` 中创建一个 `ctk.CTkToplevel`，该窗口**不继承** `root` 的图标，需要单独设置。

**修复**: 在 `InitialInterface.show()` 中创建 Toplevel 后，调用 `_set_window_icon(win)` 为其设置图标（逻辑与 `app.py` 完全一致：`iconphoto` 先 → `iconbitmap` 后）。

| 文件 | 改动 |
|------|------|
| `tcp_chat/ui/initial_interface.py:64-119` | 新增 `_set_window_icon(win)` 方法 |
| `tcp_chat/ui/initial_interface.py:1-13` | 新增 `import os`, `import tkinter as tk`, `from ..config import get_app_root` |

### 问题 3（已修复）：ICO 与 PNG 非同一张图

**现象**: 用调试方案确认 `iconbitmap()` 调用成功且无异常，但窗口图标视觉上仍然不对。

**根因**: `TCP-Chat.ico` 和 `TCP-Chat.png` 的 MD5 不同，ICO 中嵌入的 5 个尺寸（16/24/32/48/64）全部来自旧图，与 PNG 完全无关。

**修复**: 用 Pillow 从 `TCP-Chat.png` 重新生成 ICO：

```python
from PIL import Image
img = Image.open("TCP-Chat.png")
img.save("TCP-Chat.ico", format="ICO",
         sizes=[(16,16), (24,24), (32,32), (48,48), (64,64), (256,256)])
```

验证：提取 ICO 中 64×64 嵌入 PNG 与源文件逐像素对比，确认一致。

## 10. 调试方案（用于持续验证）

### 原理

`print()` 在 GUI 应用中可能被 stdout 缓冲吞掉（尤其通过管道启动时）。改用两步检测：

### 检测方式

| 输出 | 位置 | 含义 |
|------|------|------|
| `ICON_OK` | 控制台终端 | `iconbitmap()` 成功加载 ICO |
| `ICON_FAIL: <错误>` | 控制台终端 | `iconbitmap()` 抛出异常 |
| `debug_icon.log` | 项目根目录 | 每步详细日志（路径、文件存在性、异常、延迟状态） |

### 日志格式

```
[HH:MM:SS.ffffff] === _set_window_icon 被调用 ===
[HH:MM:SS.ffffff] → after(100, _apply) 已注册
[HH:MM:SS.ffffff] app_root = E:\WorkSpace\...
[HH:MM:SS.ffffff] PNG 存在? True  path=...\TCP-Chat.png
[HH:MM:SS.ffffff] iconphoto(PNG) ✅
[HH:MM:SS.ffffff] ICO 存在? True  path=...\TCP-Chat.ico
[HH:MM:SS.ffffff] iconbitmap(ICO) ✅
[HH:MM:SS.ffffff] [500ms] 窗口存在? True  hwnd=0x...
[HH:MM:SS.ffffff] [1500ms] 窗口存在? True  hwnd=0x...
[HH:MM:SS.ffffff] [3000ms] 窗口存在? True  hwnd=0x...
```

### 使用步骤

1. `find . -path "*/__pycache__*" -delete`（清缓存）
2. `python main.py`
3. 看终端有没有 `ICON_OK`
4. 看项目根目录 `debug_icon.log` 的详细步骤
5. 测试完后删除 `_set_window_icon` 中的调试代码

## 11. 额外注意：ICO 与 PNG 一致性

如果 `TCP-Chat.ico` 不是从 `TCP-Chat.png` 转换而来，需先用 pillow 重新生成：

```python
from PIL import Image
img = Image.open("TCP-Chat.png")
img.save("TCP-Chat.ico", format="ICO",
         sizes=[(256,256), (64,64), (48,48), (32,32), (16,16)])
```

确保窗口图标和 EXE 图标显示的是同一幅图。
